# 1 树模型基本概念

树模型本质是一个递归的条件判断结构：

if x1 < 5:
    if x2 > 3:
        → 类别 A
    else:
        → 类别 B
else:
    → 类别 C

树模型的基本结构

一棵树包含：

1️⃣ 根节点（Root）数据开始的地方

2️⃣ 内部节点（Decision Node）做特征划分（比如：x < 3）

3️⃣ 叶子节点（Leaf）输出结果（分类 or 回归值）

决策树模型是一类算法的集合，分为分类树和回归树两种，分类树对离散变量做决策树，回归树对连续变量做决策树。

一棵决策树的生成过程主要分为以下3个部分：

特征选择： 是指从训练数据中众多的特征中选择一个特征作为当前节点的分裂标准，如何选择特征有着很多不同量化评估标准标准，从而衍生出不同的决策树算法。

决策树生成： 根据选择的特征评估标准，从上至下递归地生成子节点，直到数据集不可分则停止决策树停止生长。 树结构来说，递归结构是最容易理解的方式。

决策树剪枝： 决策树容易过拟合，一般来需要剪枝，缩小树结构规模、缓解过拟合。剪枝技术有预剪枝（在树完全生成之前停止生成）和后剪枝（在树完全生成之后移除不必要的部分）两种。

# 2 决策树生成过程

function BuildTree(dataset, features):

    # 1️⃣ 终止条件（Conquer 的结束）
    if 所有样本属于同一类别:
        return 叶子节点(该类别)

    if features 为空:
        return 叶子节点(多数类)

    # 2️⃣ Divide：选择最优划分特征
    best_feature = 选择最优特征(dataset, features)

    # 创建当前节点
    tree = 创建节点(best_feature)

    # 获取该特征的所有取值
    for value in best_feature 的所有可能取值:

        # 划分数据集（Divide）
        subset = 从 dataset 中选出 best_feature = value 的样本

        if subset 为空:
            # 无数据，使用多数类
            child = 叶子节点(多数类(dataset))
        else:
            # 3️⃣ Conquer：递归构建子树
            child = BuildTree(subset, features - {best_feature})

        # 挂到当前节点
        tree.add_branch(value, child)

    return tree

关键在于如何选择最优特征

# 信息增益

信息熵：度量样本纯度

假定当前样本集合D中第k类样本所占比例为pk(k=1,2,...,∣y∣)p_k(k=1,2,...,|y|)pk​(k=1,2,...,∣y∣)，则D的信息熵定义为

信息越确定，包含的信息量越小，D的纯度越高

假定离散属性a有V个可能得取值{a1,a2,...,aV}\{a^1,a^2,...,a^V\}{a1,a2,...,aV}，若使用a来对样本集D进行划分，则会产生V个分支结点，其中第v个分支包含了D中所有在属性a上取值为ava^vav的样本，记为DvD^vDv

# 增益率

定义

Gain_ratio(D,a)=Gain(D,a)IV(a)IV(a)=−∑v=1V∣Dv∣∣D∣log⁡2∣Dv∣∣D∣Gain\_ratio(D,a)=\frac{Gain(D,a)}{IV(a)} \\ IV(a)=-\sum_{v=1}^V\frac{|D^v|}{|D|}\log_2\frac{|D^v|}{|D|} Gain_ratio(D,a)=IV(a)Gain(D,a)​IV(a)=−v=1∑V​∣D∣∣Dv∣​log2​∣D∣∣Dv∣​

IVIVIV (instinct value) 被称为属性 a 的固有值，属性 a 可能得取值数目越大，IV(a)IV(a)IV(a)的值通常会越大

  • 增益率 = 信息增益 ÷ 属性自身的“分裂能力”

  • IV(a)=−Σ(子集比例×log子集比例)IV(a) = - Σ (子集比例 × log 子集比例)IV(a)=−Σ(子集比例×log子集比例) “按特征 a 划分后，样本被分散的程度”

    所以如果 a 的取值数量越大，样本应该被分散的程度也会越大，IV(a)的值也会越大

    如果比例不均匀，IV 会比较小

  • 熵（IV）在“均匀分布”时最大

# C4.5决策树算法

不直接使用信息增益，而是使用增益率来选择最优划分属性

增益率准则对可取值数目较少的属性有偏好，所有 C4.5 并不是直接选择增益率最大的，而是使用启发式算法：先从候选划分属性中找到信息增益高于平均水平的属性，再从中选择增益率最高的

# 基尼指数

否则：不划分（剪枝）

# 连续值处理

在决策树中如何使用连续属性？连续属性离散化技术

假设我们用属性 a（比如年龄）来划分：

  1. 把样本按 a 的值从小到大排序

    年龄：18, 20, 25, 30, 35
标签：好, 好, 坏, 坏, 好

  2. 在相邻样本之间取“中点”：

    t₁ = (18+20)/2 = 19
t₂ = (20+25)/2 = 22.5
t₃ = (25+30)/2 = 27.5
t₄ = (30+35)/2 = 32.5

    这些都是候选划分点

  3. 逐个尝试划分

    左子集：a ≤ t
右子集：a > t

  4. 根据指标选择最优划分点

# 缺失值处理

决策树处理缺失值的核心不是“丢掉”。而是：尽量利用已有信息 + 用“概率”处理不确定性

C4.5 做法：

  1. 计算信息增益

    只用“非缺失样本”计算

    假设

    D：100个样本
其中：
- 80个有属性 a
- 20个缺失

    Step1：只用 80 个样本算信息增益

    Step2：乘一个“权重”

    权重 = 非缺失比例 = 80 / 100 = 0.8
    Gain(D, a) = 0.8 × Gain_partial

    缺失越多 → 这个特征越不可信 → Gain 被压低

  2. 划分数据

    如果样本在属性 a 上是缺失，不强行分到某一边，而是：

    按概率“分到多个分支”

    假设

    属性：天气

已知分布：
晴：50%
雨：30%
阴：20%

    对于缺失样本：

    ✔ 同时进入多个分支：
- 50% 权重 → 晴
- 30% 权重 → 雨
- 20% 权重 → 阴
